# Introduction to Agentic AI

## What is Agentic AI?
Agentic AI refers to **AI systems designed to act as autonomous agents**—capable of perceiving their environment, reasoning about tasks, 
planning actions, and executing them while learning from feedback.  
Unlike traditional models that only provide outputs when prompted, agentic systems can **initiate actions, adapt dynamically, and use external tools or APIs** 
to achieve goals.
As per Google documentation: Agentic AI is an advanced form of artificial intelligence focused on autonomous decision-making and action. Unlike traditional AI, which primarily responds to commands or analyzes data, agentic AI can set goals, plan, and execute tasks with minimal human intervention. This emerging technology has the potential to revolutionize various industries by automating complex processes and optimizing workflows. 

## Why Agents Matter in Modern AI Systems
- **Autonomy**: Agents reduce human intervention by making decisions independently.  
- **Scalability**: They can handle complex workflows across multiple systems.  
- **Integration**: Agents can connect with APIs, databases, and cloud services to perform real-world tasks.  
- **Efficiency**: By reflecting on past actions, agents improve performance over time.  
- **Industry relevance**: From customer support bots to workflow automation, agentic AI is reshaping how organizations operate.

## Difference Between Traditional ML Models and Agentic Systems
| Aspect                  | Traditional ML Models                  | Agentic AI Systems                          |
|--------------------------|----------------------------------------|---------------------------------------------|
| **Interaction**          | Passive – respond only to inputs       | Active – initiate and manage tasks           |
| **Scope**                | Narrow – single prediction/classification | Broad – multi-step reasoning and execution |
| **Adaptability**         | Static – fixed outputs                 | Dynamic – learn and adjust from feedback     |
| **Tool Use**             | Limited to predefined tasks            | Can call APIs, databases, and external tools |
| **Architecture**         | Model-centric                          | Workflow-centric (perception → reasoning → action → reflection) |

## Agent Workflow
[ Perception ] ---> [ Reasoning ] ---> [ Action ] ---> [ Reflection ]
^                                                   |
|---------------------------------------------------|


### Explanation
- **Perception**: The agent gathers input (data, environment signals, user queries).  
- **Reasoning**: It analyzes context, sets goals, and plans steps.  
- **Action**: Executes tasks, calls APIs, interacts with tools.  
- **Reflection**: Evaluates outcomes, learns from feedback, and improves future decisions.  

This loop shows how agentic systems differ from traditional one-shot ML models—they operate in a **continuous cycle of learning and acting**.

## Agent Workflow Mind Map

Agentic AI systems operate in a continuous loop:

Perception  
├── Data (structured/unstructured)  
├── Queries (user input, prompts)  
├── Sensors (environment signals, logs)  
└── Context (history, state, metadata)  

Reasoning  
├── Goal setting  
├── Planning (multi-step tasks)  
├── Knowledge retrieval (databases, APIs)  
└── Decision-making (choosing best action)  

Action  
├── APIs (Google Cloud services, external tools)  
├── Workflows (automation pipelines)  
├── Cloud services (BigQuery, Cloud Storage, Vertex AI)  
└── User interaction (responses, outputs)  

Reflection  
├── Feedback (success/failure signals)  
├── Error handling (retry, adjust strategy)  
├── Learning (improve future performance)  
└── Adaptation (refine goals, update context)

## Generative AI vs Agentic AI

| Aspect                | Generative AI                                                                 | Agentic AI                                                                 |
|------------------------|-------------------------------------------------------------------------------|----------------------------------------------------------------------------|
| **Core Purpose**       | Create new content (text, images, code, music) based on prompts.              | Orchestrate autonomous agents that use LLMs to perform actions and achieve goals. |
| **Primary Capability** | Content generation and transformation.                                        | Reasoning, planning, tool use, and execution in real systems.              |
| **LLM Role**           | Central – the model itself produces outputs.                                  | Functions as the “brain” guiding agents, often combined with external tools/APIs. |
| **Interaction Style**  | Prompt → Output (one‑shot or chained).                                        | Continuous loop: Perception → Reasoning → Action → Reflection.             |
| **Scope**              | Narrower – focused on creative or functional outputs.                        | Broader – integrates multiple steps, systems, and feedback mechanisms.     |
| **Examples**           | Generate marketing copy, write code snippets, create images.                  | Deploy marketing campaigns, query databases, automate workflows, adjust strategies. |
| **Academic Framing**   | Seen as a branch of AI focused on generative modeling.                        | Seen as part of agent‑based AI, emphasizing autonomy and multi‑agent collaboration. |
| **Vendor Perspectives**| Often highlighted by cloud providers as content creation tools.               | Framed as orchestration and workflow automation (Google, OpenAI, Anthropic, etc.). |

The below definition is adapted from Google Cloud documentation. 
While both agentic AI and generative AI are forms of artificial intelligence and can be used together, they have distinct functionalities. 

Generative AI, as its name suggests, is focused on the creation of new content, such as text, images, code, or music, based on input prompts. The LLM is at the heart of generative AI, and the value is generated by what the model can do and simple extensions of the LLM's capabilities. For example, you can generate or edit content, and even perform simple function calling and chain together various options. 

Agentic AI is a subset of generative AI that is centered around the orchestration and execution of agents that use LLMs as a "brain" to perform actions through tools. Agentic AI goes beyond content creation and function calling by executing actions in underlying systems to achieve higher-level goals. 

For example, generative AI could be used to create marketing materials, while agentic AI could then be used to deploy these materials, track their performance, and automatically adjust the marketing strategy based on the results. In this way, agentic AI can use generative AI as a tool to achieve its goals.

## Agentic AI versus AI agents
While the terms "agentic AI" and "AI agents" are often used together, there is a subtle difference. AI agents are the building blocks of agentic AI. Think of AI agents as individual tools in a toolbox, while agentic AI is the coordinated use of those tools to build an entire house. 

While an AI agent might focus on a specific task, agentic AI employs multiple agents to handle complex workflows. Agentic AI acts as an overarching system that coordinates and manages these agents to achieve broader objectives.
