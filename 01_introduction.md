# Introduction to Agentic AI

## What is Agentic AI?
Agentic AI refers to **AI systems designed to act as autonomous agents**—capable of perceiving their environment, reasoning about tasks, 
planning actions, and executing them while learning from feedback.  
Unlike traditional models that only provide outputs when prompted, agentic systems can **initiate actions, adapt dynamically, and use external tools or APIs** 
to achieve goals.

## Why Agents Matter in Modern AI Systems
- **Autonomy**: Agents reduce human intervention by making decisions independently.  
- **Scalability**: They can handle complex workflows across multiple systems.  
- **Integration**: Agents can connect with APIs, databases, and cloud services to perform real-world tasks.  
- **Efficiency**: By reflecting on past actions, agents improve performance over time.  
- **Industry relevance**: From customer support bots to workflow automation, agentic AI is reshaping how organizations operate.

## Difference Between Traditional ML Models and Agentic Systems
| Aspect                  | Traditional ML Models                  | Agentic AI Systems                          |
|--------------------------|----------------------------------------|---------------------------------------------|
| **Interaction**          | Passive – respond only to inputs       | Active – initiate and manage tasks           |
| **Scope**                | Narrow – single prediction/classification | Broad – multi-step reasoning and execution |
| **Adaptability**         | Static – fixed outputs                 | Dynamic – learn and adjust from feedback     |
| **Tool Use**             | Limited to predefined tasks            | Can call APIs, databases, and external tools |
| **Architecture**         | Model-centric                          | Workflow-centric (perception → reasoning → action → reflection) |

## Agent Workflow
[ Perception ] ---> [ Reasoning ] ---> [ Action ] ---> [ Reflection ]
^                                                   |
|---------------------------------------------------|


### Explanation
- **Perception**: The agent gathers input (data, environment signals, user queries).  
- **Reasoning**: It analyzes context, sets goals, and plans steps.  
- **Action**: Executes tasks, calls APIs, interacts with tools.  
- **Reflection**: Evaluates outcomes, learns from feedback, and improves future decisions.  

This loop shows how agentic systems differ from traditional one-shot ML models—they operate in a **continuous cycle of learning and acting**.

## Agent Workflow Mind Map

Agentic AI systems operate in a continuous loop:

Perception  
├── Data (structured/unstructured)  
├── Queries (user input, prompts)  
├── Sensors (environment signals, logs)  
└── Context (history, state, metadata)  

Reasoning  
├── Goal setting  
├── Planning (multi-step tasks)  
├── Knowledge retrieval (databases, APIs)  
└── Decision-making (choosing best action)  

Action  
├── APIs (Google Cloud services, external tools)  
├── Workflows (automation pipelines)  
├── Cloud services (BigQuery, Cloud Storage, Vertex AI)  
└── User interaction (responses, outputs)  

Reflection  
├── Feedback (success/failure signals)  
├── Error handling (retry, adjust strategy)  
├── Learning (improve future performance)  
└── Adaptation (refine goals, update context)
